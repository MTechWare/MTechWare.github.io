Overview

WinTool is a comprehensive Windows system management suite designed for IT professionals, system administrators, and power users. Built with Electron and modern web technologies, it provides a unified interface for managing Windows systems, monitoring performance, and accessing administrative tools.

Quick Start

Get started with WinTool in just a few steps:

  1. Download the portable executable from the releases page
  2. Run the application - no installation required
  3. Explore the eight core management tools
  4. Customize settings to match your workflow

System Requirements

  • Windows 10 or Windows 11 (64-bit)
  • Minimum 4GB RAM (8GB recommended)
  • 60MB free disk space
  • Administrator privileges (recommended for full functionality)
  • .NET Framework 4.7.2 or later

Installation

WinTool is designed to run without installation. Simply download and execute the portable application file to get started immediately.

Option 1: PowerShell Installation (Recommended)

Run the following command in an elevated PowerShell window:

irm https://raw.githubusercontent.com/MTechWare/wintool/refs/heads/main/WinTool_Installer.ps1 | iex

Option 2: Manual Download

  1. Visit the GitHub Releases page
  2. Download the latest WinTool.exe
  3. Run the executable file.
  4. The application will open at 60% of your screen size by default.

Option 3: Build from Source

For developers who want to build from source:

# Clone the repository
git clone https://github.com/MTechWare/wintool.git
cd wintool-simple

# Install dependencies
npm install

# Run in development mode
npm start

# Build portable executable
npm run dist

Architecture

WinTool is built using Electron, combining web technologies with native system access to create a powerful and flexible Windows management application.

Technology Stack

  • Electron: Cross-platform desktop application framework
  • Node.js: JavaScript runtime for system operations
  • HTML/CSS/JavaScript: Modern web technologies for the user interface
  • Native APIs: Windows system APIs for hardware and system access

Application Structure

Main Process

  • main.js: Electron main process entry point
  • System APIs: Native system access and hardware monitoring
  • IPC Handlers: Inter-process communication with renderer
  • Window Management: Application window creation and control

Renderer Process

  • index.html: Main application interface
  • Tab System: Modular tab-based architecture
  • UI Components: Reusable interface elements
  • State Management: Application state and data handling

Tab Architecture

Each tab is implemented as a self-contained module with its own:

  • config.json: Tab metadata and configuration
  • index.html: Tab-specific HTML content
  • styles.css: Tab-specific styling (optional)
  • script.js: Tab-specific JavaScript logic (optional)

Security Model

  • Context Isolation: Secure separation between main and renderer processes
  • Preload Scripts: Controlled API exposure to renderer process
  • Input Validation: Sanitization of user inputs and system commands
  • Privilege Escalation: Controlled elevation for administrative operations

API Reference

Service Management API

window.electronAPI.getServices()

Retrieves the list of Windows services with their current status.

Returns:

Promise<Service[]> - Array of service objects

window.electronAPI.controlService(serviceName, action)

Controls a Windows service (start, stop, restart, pause).

Parameters:
  • serviceName (string) - Name of the service to control
  • action (string) - Action to perform: 'start', 'stop', 'restart', 'pause'
Returns:

Promise<OperationResult> - Operation success/failure result

Parameters:
  • command (string) - Winget command to execute (install, uninstall, search, etc.)

  • command (string) - Chocolatey command to execute (install, uninstall, search, etc.)

Promise<CommandResult> - Command execution result with output

  • command (string) - Winget command (install, uninstall)
  • packageId (string) - Package identifier
  • args (array) - Additional command arguments
  • progressCallback (function) - Progress update callback
Returns:

Promise<InstallResult> - Installation result with progress updates

Launches a Windows system utility or administrative tool.

  • utilityCommand (string) - Command or utility to launch

The applications.json file serves as the central database for all software packages that can be installed through the Packages tab. This configuration file is critical for the package management functionality of WinTool.

Purpose & Functionality:

  • Defines the complete catalog of applications available for installation
  • Provides metadata for each application (name, description, version, etc.)
  • Specifies installation methods and commands for different package managers
  • Organizes applications into categories for easier navigation
  • Stores installation status and version information for installed applications

Structure & Format:

  • JSON array containing application objects with standardized properties
  • Each application object includes:
    • id: Unique identifier for the application
    • name: Display name shown in the UI
    • description: Brief explanation of the application's purpose
    • category: Grouping category (Browsers, Utilities, Development, etc.)
